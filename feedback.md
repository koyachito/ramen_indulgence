# 指針
## 結果画面関連
- 連続2回やったら同じ文言が出てつまらん
	- ランダムで出るようにしたらよい
	- パラメータ全部見せるんじゃなくランダムで表示されるように
	- パラメータ3つをランダムに選んで表示するようにする
	- 結果画面に「他の文言で赦される」ボタンを作って文言変更できるように
	- でもそもそもランダム要素いらないかも...？分岐してるから十分ゲーム性あるかも
- 最後に「アーメン」にかけて「ラーメン」といわせる
- 画像コピーがやっぱ微妙。ダウンロードするくらいならいっそ連携しちゃうほうがマシなので、「X連携して投稿する」ボタンに
- 「だし」「だし」->「赦されました」は日本語としてちょっと微妙。あなたは〇〇をされました。今日は〇〇です。よって…　家系一杯が赦されました。ラーメン...と

- それぞれの質問で一番良くない選択肢を選び続けたら鬼の形相のシスターを出す: angry.pngとして追加済み。要背景透過
	- ショート動画をずっと見てた->二郎系or家系
- 近くのラーメンを探すボタンをもっと上に持ってくる -> 「赦されました」の下にXとラーメン検索を持ってくる
- 「今日は寝ろ」をめっちゃ笑顔にする。実現条件をもっと簡単にする。連続3回鬼の形相したら診断押すだけでそっちに飛んで、3時間は治らないとか
- 選択したラーメンのあるあるを免罪結果に表示する

## 質問画面関連
- 質問はもうちょっと多いほうがいい、懺悔感が出る
- 質問追加：ラーメンを食べない理由は？->「ある」「ない」を選ぶ
- 「今日なしとげたこと」関連
	- 「この内容で次へ」をなくす。複数選択ではなく一個だけ選ぶように変える
	- なしとげたことが二択なのがよくない　何もしてない人にはこじつけで許可すること
	-「何もしていない」を選んだら結果画面で「これから何かをするために、原動力となる一杯をどうぞ」が出るように
	- 今日何やったか聞くところにもっと選択しあってもいい、いつもよりちょっと早く起きたとか、家事したとか
	- おふざけ選択肢あってもいい、ショート動画・タイムラインをずっと見てたとか、推しの配信を頑張って何時間もみてたとか
	- ちょっとだけいいことやってるからもしかしたら救われるかも...みたいな
- 言い訳を選ばせるのがあってもいい
	- 「どんなふうに赦されたいですか？」「褒めてほしい」「甘やかしてほしい」「厳しくてもいい」
	- 「厳しくてもいい」&「3食以上」&「ネットサーフィン」で鬼フラグ。
	- 鬼フラグを3時間以内に3回立てたら「今日は寝ろ」
- 1個目の質問は「今日すでに何食食べましたか？」がよい。現状だと夜前提の質問に -> 質問の順番も入れ替える
- 3食以上選んだときのみ「そのうちラーメンは何回でしたか？」
	- 2食以上だったら「体調は大丈夫ですか...?」とともにシスターの画像を出す->診断に戻る（結果には影響しない）:worry.png追加済み。要背景透過
- スマホだと「その他」のラーメンが下に隠れて出ない、余白調整
- その他を押すとランダムにおすすめのラーメンを結果画面で表示する

## その他・全般
- 統計タブを出すのは結果以降。トップページと診断中には出さない
- データを一時的にではなく保存したいのでposgresqlへの積み替え
	- 累計診断数を出す
- 鬼フラグ管理のためにCookie取得


# 具体指示
- 全質問を一気に並べるフォームではなく、1問ずつ進む会話ステップ式にする
- 基本フローは「シスターの質問 → ユーザーの回答 → シスターのリアクション(画像つき） → 次の質問」
- 結果画面の文章は、自然な「審議 → 赦し」形式にする
- 同じ入力でも結果文が毎回少し変わるように、文言パーツをランダム化する
- 結果画面では「近くのラーメンを探す」ボタンを「X連携して投稿する」ボタンより上に置く
- X投稿は、生成した結果画像をそのまま投稿する導線にする
- 統計タブは、トップページや診断中には表示せず、結果画面以降のみ表示する
- スマホで選択肢やボタンが隠れないようにする
- Cookie を使って、鬼結果が短時間に続いた場合だけ「今日は寝ろ」結果に寄せる
- 「今日は寝ろ」は現状の大きい赤文字表示を維持する
- データ保存は、可能ならRenderの有料PostgreSQLへ載せ替える

============================================================
事情聴取画面：会話ステップ式UI
============================================================

診断画面は、1問ずつ進む会話ステップ式にしてください。

基本フロー：

1. シスターが質問する
2. ユーザーが選択肢を選ぶ
3. シスターが回答内容に応じてリアクションする
4. 「次へ」ボタンで次の質問へ進む
5. 最後の質問のあと「診断する」ボタンを表示する

画面構成：

- シスター画像
- シスターのセリフ
- 現在の質問文
- 選択肢
- 次へボタン
- 戻るボタン
- 最終ステップのみ診断するボタン

挙動：

- シスターを常時フロート表示しない
- 質問ごとにシスター画像とコメントを切り替える
- ユーザーが回答した直後にリアクション文を表示する
- リアクション表示後に次の質問へ進める
- スマホで見やすいように、1画面1質問を基本にする
- 選択肢やボタンが画面下に隠れないようにする
- 戻るボタンで前の質問に戻れる
- 戻った場合、既存回答があれば選択状態を復元する

============================================================

oni_flag は、危険または懺悔が必要な回答の総合判定に使います。

oni_flag 加算条件：

- meals = 4
- ramen_count_today >= 2
- achievement = shorts
- reason_not_to_eat = ignore
- ramen_type = jiro
- forgiveness_style = strict
- after_plan = more_shorts

mood = angry は、ユーザーがむしゃくしゃしている状態として扱います。
confession_score は加算して構いません。
oni_flag は加算しません。

============================================================
Cookie 管理仕様
============================================================

目的：

3時間以内に ogre 結果が3回出た場合、しばらく「今日は寝ろ」結果に寄せるため、Cookieを使います。

Cookie項目：

- oni_count
- oni_last_at
- sleep_until

oni_count を増やす対象：

- 結果画面で result_type = ogre になった場合

挙動：

診断結果で result_type = ogre になった場合：

- 現在時刻 - oni_last_at <= 3時間
  → oni_count += 1

- 現在時刻 - oni_last_at > 3時間
  → oni_count = 1

- oni_last_at = 現在時刻

oni_count >= 3 になった場合：

- sleep_until = 現在時刻 + 3時間

============================================================
「今日は寝ろ」結果仕様
============================================================

表示条件：

以下のどちらかに該当した場合、通常の診断結果よりも「今日は寝ろ」結果を優先してください。

条件1：

sleep_until が現在時刻より未来

条件2：

現在時刻が深夜
かつ
meals >= 3
かつ
after_plan が sleep または more_shorts

深夜の目安：

- current_hour >= 23
- または current_hour < 5

表示画像：

prayer.png

タイトル：

今日は寝ろ

表示：

- 現状の大きい赤文字表示を維持してください
- 既存の「今日は寝ろ」表示スタイルがある場合は、それを優先してください

「今日は寝ろ」中でも、診断フォーム自体は使えるようにしてください。
sleep_until が切れるまでは、結果画面は「今日は寝ろ」に寄せてください。
完全ロックはしないでください。

============================================================
結果画面：文章構造
============================================================

結果文の基本構造：

あなたは今日、{achievement_reason}。
さらに、{mood_reason}という事情があります。
本来であれば、{caution_reason}。

しかし、{forgiveness_reason}。

よって本日、
{ramen_type_label}一杯は赦されました。

ラーメン。

最後は必ず「ラーメン。」で終えてください。
これは「アーメン」にかけた締めです。

============================================================
結果文のランダム化
============================================================

同じ入力でも毎回同じ文言にならないように、以下のパーツをランダムに選んでください。

- achievement_reason
- mood_reason
- caution_reason
- forgiveness_reason

完全ランダムで構いません。
ただし、回答内容と矛盾する文言は出さないでください。

例：

achievement = worked の場合：

- 労働または学習という尊い務めを果たしました
- 今日やるべきことに向き合いました
- 少なくとも、何かを前に進めました

achievement = nothing の場合：

- 何もしていないという事実を正直に申告しました
- 今日はまだ、力を温存しています
- これから何かをする可能性を残しています

achievement = shorts の場合：

- 時間を電子の海に捧げました
- タイムラインの巡礼を終えました
- 短い動画を無数に見届けました

mood = tired の場合：

- 心身に疲労が見られます
- 休息と塩分を求めているようです

mood = hungry の場合：

- 空腹という重大な証言があります
- 胃袋が正式に申し立てを行っています

mood = lonely の場合：

- 温かいものを必要としているようです
- 丼のぬくもりが必要な状態です

caution_reason は danger_score に応じて変えてください。

danger_score <= 1:

- 特に大きな問題は見当たりません
- 審議は比較的穏やかに進みました

danger_score >= 2:

- 慎重な判断が必要です
- 罪の気配がないとは言えません

danger_score >= 4:

- 通常であれば止めるべき案件です
- かなり厳しい審議となりました

forgiveness_reason は forgiveness_style に応じて変えてください。

praise:

- それでも、今日を乗り切ったあなたには報いが必要です
- あなたはよくやっています。温かい一杯を受け取ってください

spoil:

- 今日はもう、ラーメンに甘やかされてもよいでしょう
- 大丈夫です。スープに抱きしめられてください

strict:

- 反省は必要です。しかし、反省とラーメンは両立します
- 罪はあります。ですが、赦しの余地もあります

push:

- 迷っている時間が一番よくありません。行きましょう
- 暖簾は、くぐる者の前にだけ現れます

oracle:

- 麺は示され、スープは満ちています
- 丼は沈黙し、しかし確かにあなたを呼んでいます

============================================================
X連携して投稿するボタン仕様
============================================================

ボタン文言：

X連携して投稿する

目的：

結果画面で生成したラーメン免罪符画像を、そのままXに投稿しやすくすること。

想定挙動：

- 結果画面のカードを画像として生成する
- 生成画像には、診断タイトル、シスター画像、赦し本文、ラーメン種別、発行日時などを含める
- ユーザーがその画像をXに投稿できる導線を用意する

実装方針：

1. 可能なら、結果カードを画像化する
   - html2canvas などを使用してよい
   - サーバー側で画像生成してもよい
   - 既存の画像生成処理がある場合は、それを利用する

2. Xの仕様上、Web Intent URLだけではローカル生成画像を自動添付できない場合があります。
   その場合は以下の導線にしてください。

   - 画像を生成
   - 画像をダウンロード、またはコピーできるようにする
   - X投稿画面を開く
   - 投稿文をセットする
   - ユーザーが生成画像を添付して投稿できるようにする

3. 画像をサーバーに保存し、公開URLを発行できる構成にできる場合は、その画像URLを投稿文に含めてください。

投稿文例：

ラーメン免罪符が発行されました。

本日、{ramen_type_label}一杯は赦されました。

ラーメン。

#ラーメン免罪符

X投稿URL：

https://twitter.com/intent/tweet?text={encoded_text}
