
## 理由生成ロジック

### 概要

本アプリでは、ユーザーが入力した情報とブラウザから取得できる情報をもとに、ラーメンを食べてよい理由を一行ずつ生成する。

理由文は、以下の要素から構成する。

```text
時間帯理由
+ 食事回数理由
+ 運動理由
+ 日付理由
+ 結論
```

最終的な文章は、複数の理由を「〜だし、」で積み上げたあと、最後に「だからラーメン食べていいです！」で締める。

### 設計方針

理由生成には、外部APIや乱数を使わない。

使用するデータは、すでに取得済み、またはユーザーが入力済みのものに限定する。

使用するデータは以下。

```text
- 現在時刻
- 今日の食事回数
- 今日8000歩以上歩いたかどうか
- 今日の日付
```

天気、ニュース、位置情報などは取得しない。
それらを理由生成のためだけに追加すると、実装コスト・権限許可・API依存が増え、アプリの軽さを損なうためである。

### 理由文の基本構造

生成される文章の例は以下。

```text
もう23時なので、理性よりもラーメンを優先すべきだし、
今日はまだ2食しか食べてないし、
今日は8000歩以上歩いてるし、
今日は6月16日だから「特に何もない日。つまりラーメンで意味を与える余地がある」。

だからラーメン食べていいです！
```

画面上では、文章形式または理由リスト形式で表示する。

文章形式：

```text
もう23時なので、理性よりもラーメンを優先すべきだし、
今日はまだ2食しか食べてないし、
今日は8000歩以上歩いてるし、
今日は6月16日だから「特に何もない日。つまりラーメンで意味を与える余地がある」。

だからラーメン食べていいです！
```

理由リスト形式：

```text
ラーメン許可理由

・もう23時なので、理性よりもラーメンを優先すべきだし
・今日はまだ2食しか食べてないし
・今日は8000歩以上歩いてるし
・今日は6月16日だから「特に何もない日。つまりラーメンで意味を与える余地がある」

だからラーメン食べていいです！
```

### 時間帯理由

現在時刻をもとに、時間帯ごとの理由文を生成する。

```text
7:00以上 11:00未満
一日の始まりにはラーメンを食べるべきだし、

11:00以上 14:00未満
昼時にラーメンを食べて午後のエネルギーを蓄えるべきだし、

14:00以上 18:00未満
おやつのラーメンはなによりも元気の源だし、

18:00以上 20:00未満
夕飯のラーメンは一日の疲れをとってくれるし、

20:00以上 または 7:00未満
もう{hour}時なので、理性よりもラーメンを優先すべきだし、
```

注意点として、20時ちょうどが条件から漏れないようにする。

そのため、夜・深夜帯の条件は以下とする。

```ts
hour >= 20 || hour < 7
```

### 食事回数理由

ユーザーが入力した今日の食事回数をもとに、理由文を生成する。

```text
0食
今日はまだ何も食べてないし、

1〜2食
今日はまだ{foodTimes}食しか食べてないし、

3食以上
{foodTimes}食食べたあとでもラーメンは別腹だし、
```

食事回数が何回であっても、最終的にはラーメンを肯定する。
ただし、理由文のトーンは食事回数によって変える。

### 運動理由

ユーザーが「今日8000歩以上歩いた」と入力した場合のみ、運動理由を追加する。

```text
今日は8000歩以上歩いてるし、
```

歩いていない場合は、運動理由を追加しない。

歩いていないユーザーを責める文言は表示しない。
このアプリは健康管理アプリではなく、ラーメンを食べる理由を発行するジョークアプリであるため、説教感を出さない。

### 日付理由

現在の日付をもとに、日付理由を生成する。

日付は `MM-DD` 形式のキーに変換し、日付ごとの理由辞書から文言を取得する。

例：

```ts
const dateEvents = {
  "01-01": "元日。新年最初の麺は縁起がいい",
  "01-07": "七草粥の日。草だけでは心が足りない",
  "02-03": "節分。豆だけでは腹が満たされない",
  "02-14": "バレンタインデー。チョコの後には塩分が必要",
  "04-01": "エイプリルフール。食べないという嘘はつかなくていい",
  "07-07": "七夕。天の川よりスープの川を渡るべき日",
  "10-31": "ハロウィン。かぼちゃより麺を信じる夜",
  "11-11": "麺が4本立っているように見える日",
  "12-24": "クリスマスイブ。聖なる夜には聖なる麺を",
  "12-31": "大晦日。一年の締めには細く長いものが必要",
};
```

該当する日付理由が存在しない場合は、デフォルト文言を使用する。

```text
特に何もない日。つまりラーメンで意味を与える余地がある
```

生成文の形式は以下。

```text
今日は{month}月{day}日だから「{event}」。
```

例：

```text
今日は11月11日だから「麺が4本立っているように見える日」。
```

### 結論文

すべての理由文を表示したあと、最後に必ず以下の結論文を表示する。

```text
だからラーメン食べていいです！
```

このアプリでは、入力内容にかかわらず、最終的にはラーメンを食べることを肯定する。
ただし、医学的・栄養学的な判断ではなく、あくまでジョークとして肯定する。

### TypeScript実装イメージ

```ts
type FoodTimes = 0 | 1 | 2 | 3 | 4 | 5;

const dateEvents: Record<string, string> = {
  "01-01": "元日。新年最初の麺は縁起がいい",
  "01-07": "七草粥の日。草だけでは心が足りない",
  "02-03": "節分。豆だけでは腹が満たされない",
  "02-14": "バレンタインデー。チョコの後には塩分が必要",
  "04-01": "エイプリルフール。食べないという嘘はつかなくていい",
  "07-07": "七夕。天の川よりスープの川を渡るべき日",
  "10-31": "ハロウィン。かぼちゃより麺を信じる夜",
  "11-11": "麺が4本立っているように見える日",
  "12-24": "クリスマスイブ。聖なる夜には聖なる麺を",
  "12-31": "大晦日。一年の締めには細く長いものが必要",
};

function getMonthDay(date: Date): string {
  const month = String(date.getMonth() + 1).padStart(2, "0");
  const day = String(date.getDate()).padStart(2, "0");

  return `${month}-${day}`;
}

function getTimeReason(hour: number): string {
  if (hour >= 7 && hour < 11) {
    return "一日の始まりにはラーメンを食べるべきだし、";
  }

  if (hour >= 11 && hour < 14) {
    return "昼時にラーメンを食べて午後のエネルギーを蓄えるべきだし、";
  }

  if (hour >= 14 && hour < 18) {
    return "おやつのラーメンはなによりも元気の源だし、";
  }

  if (hour >= 18 && hour < 20) {
    return "夕飯のラーメンは一日の疲れをとってくれるし、";
  }

  return `もう${hour}時なので、理性よりもラーメンを優先すべきだし、`;
}

function getFoodReason(foodTimes: FoodTimes): string {
  if (foodTimes < 1) {
    return "今日はまだ何も食べてないし、";
  }

  if (foodTimes >= 1 && foodTimes <= 2) {
    return `今日はまだ${foodTimes}食しか食べてないし、`;
  }

  return `${foodTimes}食食べたあとでもラーメンは別腹だし、`;
}

function getWalkedReason(walked: boolean): string | null {
  if (walked) {
    return "今日は8000歩以上歩いてるし、";
  }

  return null;
}

function getDateReason(date: Date): string {
  const key = getMonthDay(date);
  const event =
    dateEvents[key] ??
    "特に何もない日。つまりラーメンで意味を与える余地がある";

  return `今日は${date.getMonth() + 1}月${date.getDate()}日だから「${event}」。`;
}

function buildRamenPermissionText(
  foodTimes: FoodTimes,
  walked: boolean,
  date: Date = new Date()
): string {
  const hour = date.getHours();

  const reasons: string[] = [
    getTimeReason(hour),
    getFoodReason(foodTimes),
  ];

  const walkedReason = getWalkedReason(walked);
  if (walkedReason !== null) {
    reasons.push(walkedReason);
  }

  reasons.push(getDateReason(date));
  reasons.push("");
  reasons.push("だからラーメン食べていいです！");

  return reasons.join("\n");
}
```

入力：


出力：

```text

```

### X投稿文への利用

生成した理由文は、そのままX投稿文にも利用できる。

投稿文の例：

```text
ラーメン許可証 発行完了

もう23時なので、理性よりもラーメンを優先すべきだし、
今日はまだ2食しか食べてないし、
今日は8000歩以上歩いてるし、
今日は6月16日だから「特に何もない日。つまりラーメンで意味を与える余地がある」。

だからラーメン食べていいです！

#ラーメン許可証
```

X投稿時は文字数制限があるため、必要に応じて理由文を短縮する。

短縮版では、以下の優先順位で理由を残す。

```text
1. 時間帯理由
2. 食事回数理由
3. 日付理由
4. 運動理由
```

ただし、運動理由はユーザーにとって肯定感が強いため、実装時に優先順位を調整してもよい。
