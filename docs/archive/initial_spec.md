ラーメン免罪符：現時点の仕様まとめ
1. コンセプト
深夜・疲労時・勉強後などに「ラーメン食べたい」という欲望を、ユーモア込みで正当化するWebアプリ
ユーザーに軽い質問をして、ラーメンを食べてもよいかどうかを診断する
最終的に「免罪符」っぽい診断文を出す
罪悪感を減らしつつ、ちょっと笑える体験にする
ガチ健康管理アプリではなく、ネタ・セルフケア・SNS共有向けの小さなWebアプリ
2. 想定ユーザー
勉強・仕事・バイト後にラーメンを食べたくなる人
深夜に外食欲が出る人
「今日は食べてもいい理由」がほしい人
Xなどに診断結果を投稿したい人
42Tokyo / エンジニア学習者 / オタク・ネット文化圏にも刺さる軽いネタアプリ
3. MVPの目的
企画からリリースまでの一連の流れをポートフォリオ化する
「自分で作って公開したWebアプリ」として見せられる状態にする
技術的には、FastAPI / Python / HTML / CSS / JS / SQLiteあたりの実装経験を証明する
ただの写経ではなく、ユーザー体験・診断ロジック・公開・READMEまで含めた成果物にする

5. 診断ロジックの方向性
厳密な健康判定ではなく、スコアベースのネタ判定
入力項目ごとに加点・減点する
例：
今日あまり食べていない → 加点
運動した → 加点
長時間勉強・作業した → 加点
深夜すぎる → 減点または注意文追加
すでに食事回数が多い → 減点
疲労度が高い → 加点、ただし寝ろ判定もあり
結果は複数パターンに分ける
完全免罪
条件付き免罪
半ライス禁止付き免罪
スープ残し推奨
今日は寝ろ
明日の昼に回せ
ネタ判定だが、危険な健康助言にはしない
6. 出力文のイメージ
「本日のラーメン摂取は、情状酌量の余地あり」
「あなたは今日、十分に労働／学習を行いました」
「よって、味噌ラーメン一杯を許可します」
「ただし、スープ完飲および大盛りライスの併用は慎むこと」
「明日の自分への申し送り：水を飲んで寝る」
7. X投稿機能
結果文をXに投稿できる
ログイン連携はしない
Xの投稿URLを生成するだけ
投稿文には以下を含める
診断結果
食べたいラーメンの系統
免罪符っぽい一文
アプリURL
ハッシュタグ
例：
本日のラーメン免罪：条件付き許可
ただしスープ完飲は禁止。
#ラーメン免罪符
8. 近隣検索リンク
位置情報APIは使わない
個人情報や現在地を保存しない
Google Mapsの検索URLを生成するだけ
ユーザーが押すと、Google Mapsで「近くのラーメン」などを検索できる
系統を選んでいる場合は検索語に反映する
例：近くの 家系ラーメン
例：近くの 味噌ラーメン
9. 保存するデータ
原則、個人情報は保存しない
ログインなし
ユーザー識別なし
位置情報なし
保存するのは最低限
通算診断回数
診断結果の種類ごとのカウント

SQLiteを使う想定
ユーザーごとの履歴保存はMVPではやらない
